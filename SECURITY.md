# Security Policy

## Supported Versions

This repository is documentation-focused and does not ship runtime binaries.
Security reports are still welcome for:

- Supply chain risks in workflows.
- Malicious links or content in contributions.
- Abuse of repository automation.

## Reporting a Vulnerability

Please do not open public issues for sensitive reports.
Use private reporting through GitHub Security Advisories.

## Response Targets

- Initial acknowledgment: within 5 business days.
- Triage decision: within 10 business days.
- Public fix/update timeline: communicated after triage.
